import ballerina/io;type Address record {
    string line01;
    string line02;
    string city;
    string state;
    string zipcode;
};
type Person record {
    string name;
    int age;
    Address? addr;
};
function getPerson(string name, int age) returns Person? {
    if (name == "" || age == 0) {
        return;
    } else {
        Person p = {
            name: name,
            age: age,
            addr: {
                line01: "No. 61",
                line02: "Brandon street",
                city: "Santa Clara",
                state: "CA",
                zipcode: "95134"
            }
        };
        return p;
    }
}public function main() {
    Person? p1 = getPerson("John", 30);
    io:println(p1);
    string? city1 = p1.addr.city;
    io:println(city1);
    Person? p2 = getPerson("", 0);
    string? city2 = p2.addr.city;
    io:println(city2);
    string defaultCity = "San Jose";
    string city = city2 ?: defaultCity;
    io:println(city);
}

Nil Lifting

The Optional Type example demonstrates how to eliminate null reference errors by using optional types in Ballerina. This example demonstrates how to operate on optional typed variables. E.g., the + operator cannot operate on values of int? type. However, there are some operators that work on optional types. They are called lifted operators because they operate on optional types by lifting the operators that already exist on non-optional types.

Immutable Values

 
import ballerina/io;

type Address record {
    string line01;
    string line02;
    string city;
    string state;
    string zipcode;
};

type Person record {
    string name;
    int age;
    Address? addr;
};

According to the Person record type descriptor, the addr field could either hold an Address record or the value nil (()).

    } else {

This function optionally returns a Person value.

Define a Person record.

 
public function main() {
    Person? p1 = getPerson("John", 30);
    io:println(p1);

    string? city1 = p1.addr.city;

The field access operator is a lifted operator in Ballerina. The types of both p1 and p1.addr are optional types. If the type of either p1 or p1.addr is () at run time, () would be assigned to the city1 variable, else the string value of p1.addr.city would be assigned.

 
    io:println(city1);

The value “Santa Clara” is displayed in the console.

 
    Person? p2 = getPerson("", 0);

Invoking getPerson() as follows will result in () being assigned to p2.

 
    string? city2 = p2.addr.city;
    io:println(city2);

The city2 variable will thus contain nil as the value.

 
    string defaultCity = "San Jose";
    string city = city2 ?: defaultCity;
    io:println(city);
}

Here, the Elvis operator is used to eliminate nil. If the value of the first expression is not nil, the Elvis operator returns that value. If the value of the first expression is nil, the operator returns the value of the second expression.

Error Handling

In Ballerina, errors can be returned or can cause abrupt completion via panic. This example demonstrates how errors can be returned, and how returned errors can be handled.

Define a record to represent the error details. This record can have fields of types that belong to anydata|error.

 
type AccountNotFoundError error<string, AccountNotFoundErrorData>;

User-defined error types can be introduced by specifying a reason type-descriptor and optionally a detail type-descriptor. The reason type descriptor should be a subtype of string and the detail type descriptor should be a subtype of record {} or map<anydata|error>.

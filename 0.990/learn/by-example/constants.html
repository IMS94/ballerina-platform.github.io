Constants

Constants are known values at compile time. Only value type constants (boolean, byte, decimal, float, int, string) are supported at the moment. The constants are useful for defining enumeration types.

We can use these constants just like any other value type variable except the value of these constants cannot be changed after initializing. The difference between the final variables and constants is that the value of the final variables can be initialized at runtime. However, constants must be initialized at compile time.

Integer Ranges

 
import ballerina/io;

const string GET = "GET";

Constants are defined using the const modifier.

 
const POST = "POST";

Constants can be defined without type using var keyword. Then the type is inferred from the right hand side.

 
type ACTION GET|POST;

These constants can be used to create type definitions. This type definition can only hold GET or POST values.

 
const int I = 125;

Integer constant.

 
public function main() {

    ACTION get = GET;
    testAction(get);

This is identical to the code line ACTION get = "GET";.

 
    ACTION post = "POST";
    testAction(POST);

    int value = I + 10;
    io:println(value);

We can concatenate I with any other integer value.

 
}

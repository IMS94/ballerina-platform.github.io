Constrained JSON

A JSON can be constrained with a record definition to restrict the fields of the JSON to that of the record. If you try to access a field with an invalid field as per the record definition or initialize a JSON with invalid fields, this will cause a compilation error. Restricting the fields of the JSON will enforce the format of the JSON variable.

JSON/Record/Map Conversion

 
import ballerina/io;

type Person record {
    string name;
    int age;
    string city;
    !...
};

Defining a sealed record Person.

 
public function main() {

    json<Person> person = { name: "Jon" };

Create a new JSON that is constrained by the Person record. Only the fields defined in the Person record can be accessed. If we try to access a non existing field, it will produce a compilation error.

 
    person.age = 25;
    person.city = "Colombo";
    io:println(person);

We can access fields defined in the Person record without any issue.

 
    json employee = person;

We can assign this constrained JSON to a JSON. This will allow us to add new elements that are not in the record.

 
    employee.profession = "Software Engineer";
    io:println(employee.profession);
}

After that, we can add new elements to the JSON, and access them.

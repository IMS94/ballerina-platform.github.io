Error Lifting

The error lifting operator enables you to navigate through a set of fields without worrying about error or nil values along the way. In an event of an error or a nil return, the operator stops the navigation and the value of the entire expression evaluates to error.

import ballerina/io;

type Response record {
    Status|error status;
};

type Status record {
    string message;
    int code;
};

public function main() {
    error e = error("response error");
    Response|error firstResponse = e;

    int|error statusCode1 = firstResponse!status!code;
    io:println("The status code: ", statusCode1);

Navigate the fields, by lifting the error. Using the ! operator stops the navigation if the value returned is error, and then assigns that to the statusCode1 variable.

 
    Response|error|() secondResponse = ();

Consider a scenario where the secondResponse is nil.

 
    int|error|() statusCode2 = secondResponse!status!code;
    io:println("The status code: ", statusCode2);
}

The error lifting operator lifts nil by default. If the secondResponse is nil, it stops navigating the rest of the fields and the value of the secondResponse!status!code expression evaluates to nil.

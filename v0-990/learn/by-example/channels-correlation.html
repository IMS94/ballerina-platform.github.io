Message Correlation with Channels

Channel is a constrained type that can be defined only as a top level node. It is introduced for communication between parallel processes in Ballerina programs. Channels can be used for message correlation by sending and receiving messages via different resources to the same channel. In workflow related scenarios, you can use channels to receive intermediate input. Please note that channel is an experimental feature in Ballerina.

Defines a channel with json constrained type.

 
service channelService on new http:Listener(9090) {

    resource function receive(http:Caller caller, http:Request request) {

        string key = "123";

A key can be associated with a channel action. You can use a key to correlate receivers and senders.

 
        json jsonMsg;

        jsonMsg = <- jsonChannel, key;

Receive a message from the channel with given key. Execution waits here if the message is not available.

 
        var result = caller->respond(jsonMsg);

Send the received message as the response.

 
        if (result is error) {
            log:printError("Error sending response", err = result);
        }
    }

    resource function send(http:Caller caller, http:Request request) {

        json|error message = request.getJsonPayload();

Extract message from the request.

 
        string key = "123";

Define the same key as the receiver’s key.

Send a message to the channel. One of the receivers waiting on this key receives it. If there is no receiver, the message is stored and execution continues. A receiver can arrive later and fetch the message.

 
        json jsonMessage = {};
        if (message is json) {
            jsonMessage = message;
        } else {
            log:printError("Invalid message content", err = message);
        }

        jsonMessage -> jsonChannel, key;

        var result = caller->respond({ "send": "Success!!" });
        if (result is error) {
           log:printError("Error sending response", err = result);
        }
    }
}

import ballerina/io;
channel<xml> xmlChn = new;public function main(string... args) {    worker w1 {
        map<string> keyMap = { myKey: "abc123" };
        io:println("w1 Waiting for a message...");
        xml message;
        message = <- xmlChn, keyMap;
        io:println("w1 Received message: ", message);
        io:println("w1 Waiting for a null key message...");
        message = <- xmlChn;
        io:println("w1 Received message: ", message);
    }    worker w2 {
        map<string> keyMap = { myKey: "abc123" };
        xml msg = xml `<msg><name>ballerina</name><worker>w2</worker></msg>`;
        msg -> xmlChn, keyMap;
        io:println("Sent message from w2");
    }    worker w3 {
        xml msg = xml `<msg><name>ballerina</name><worker>w3</worker></msg>`;
        msg -> xmlChn;
        io:println("Sent message from w3");
    }    wait w1;
}

Worker Interactions with Channels

You can use channels for inter-worker communication and also to synchronize workers. A channel can be considered as a common pipe that is accessible by any number of workers within the channel’s lexical scope. Please note that channel is an experimental feature in Ballerina.
